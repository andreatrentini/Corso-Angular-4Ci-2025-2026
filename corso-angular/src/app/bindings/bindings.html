String interpolation

Con la string interpolation, visualizzo nel template HTML un valore memorizzato in una "variabile" typescript.
Possono essere usate solo variabili che sono attributi della classe component.

In una applicazione zoneless per garantire cha la vista venga aggiornata devo usare i signals.

Cos'è un signal?

Un signal è una funzione wrapper che gestisce un valore di qualsiasi tipo: numeri, stringhe, array, oggetti...

E' la signal che controlla se un valore viene modificato e in tal caso marca la zona con quel valore come dirty, e quindi richiede il refresh del template HTML.

Quali gli step da seguire:
  1. Dichiarare la signal in typescript
  2. Inserire una chiamata alla funzione signal fra doppie parentesi graffe

Event binding

Per utilizzare event binding, si toglie on dal nome dell'evento e si racchiude fra parentesi tonde il nome rimanente. Il valore dell'evento DEVE essere il nome di un metodo della classe del componente.
ATTENZIONE: ricordarsi le parentesi tonde dopo il nome del metodo!!!

Property binding

Una property corrisponde ad un attributo di un elemento HTML. Ad esempio il tag img prevede un attributo obbligatorio: src="nome dell'immagine". src dal punto di vista di angular è una property.

Per usare property binding racchiudo il nome dell'attributo fra parentesi quadre e come valore assegno un'espressione typescript: il nome di una variabile, il valore di un signal, può anche essere una espressione che restituisce vero/falso.

Class binding

Si mette l'attributo class fra parentesi quadre e si assegna come volore un'espressione typescript.

Può essere usato nella modalità [class]="elenco di classi", oppure [class.nomeClasse]="boolean".
